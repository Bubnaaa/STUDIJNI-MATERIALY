 ### Router-on-a-stick
- Není uprostřed sítě, je připojen jedním trunk linkem ke switchi
- Jeden fyzický port routeru je rozdělen na více subinterfaců
- Každá VLAN má
	- Vlastní IP
	- Vlastní default gateway

### Konfigurace VLAN a trunků na S1
#### Postup:
1. Vytvoření a pojmenování VLAN
2. Vytvoření management rozhraní (SVI)
3. Nastavení access portů
4. Nastavení trunk portů

### Konfigurace subinterfaců na R1
- Použití metody router-on-a-stick
- Musí mít:
	- Encapsulation dot1Q VLAN_ID
	- ip adress -> default gateway
- Fyzické rozhraní musí být no shutdown, jinak subinterfacy nefungují

### Ověření konektivity 
- Nejprve zkontrolovat IP kofiguraci
- Test komunikace:
	- PC1->PC2 (Jiná VLAN)
	- ¨PC1 management IP switche
- Úspěšný ping mezi VLANami = Inter-VLAN routing funguje

### Ověření a troubleshooting

Používané příkazy na routeru a switchi:
- `show ip route`
    - kontrola připojených VLAN sítí (C, L)
- `show ip interface brief
    - stav rozhraní a subrozhraní   
- `show interfaces`
    - detailní stav portů    
- `show interfaces trunk`
    - ověření trunk linků

- V routovací tabulce musí být:
	- Každá VLAN jako connected
	- Správný subinterface jao výstup
