### VLAN Trunks definitions
- = L2 spoj mezi switchemi, který přenáší více VLANů současně
- Používá 802.1P tagování framů - Přidává VLAN ID do Ethernet framu
- Přes trunk se posílá všechna VLAN komunikace, pokud není omezena příkazem `allowed vlan`
- Trunk je nutný pro:
	- Komunikaci VLANů mezi switchemi
	- Připojení routeru-on-a-stick
	- Připojení k serverům s více VLANy

### Trunk Config
- Topologie:
- VLAN 10 – Faculty (172.17.10.0/24)
- VLAN 20 – Students (172.17.20.0/24)
- VLAN 30 – Guests (172.17.30.0/24)
- VLAN 99 – Native VLAN (172.17.99.0/24)
```
S1(config)# interface fa0/1
S1(config-if)# switchport mode trunk
S1(config-if)# switchport trunk native vlan 99
S1(config-if)# switchport trunk allowed vlan 10,20,30,99
S1(config-if)# end
```
- `switchport mode trunk` → zapne trunk režim.
- `switchport trunk native vlan 99` → určí VLAN bez tagu 
- `switchport trunk allowed vlan 10,20,30,99` → trunk propustí pouze uvedené VLANy.

### Ověření configu trunku
`show interfaces fa0/1 switchport
`
### Reset trunku na výchozí stav
```
S1(config)# interface fa0/1
S1(config-if)# no switchport trunk allowed vlan
S1(config-if)# no switchport trunk native vlan
S1(config-if)# end
```

### Zrušení trunku
```
S1(config)# interface fa0/1
S1(config-if)# switchport mode access
S1(config-if)# end
```
