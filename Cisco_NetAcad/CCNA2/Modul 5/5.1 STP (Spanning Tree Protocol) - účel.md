- Umožňuje redundantní spojení v síti a zabránit vzniku smyček na L2
### Důvod vzniku redundantního spojení
- Aby síť nespadla, když přestane fungovat kabel nebo port
- Hierarchický design vyžaduje víc fyzických cest k jednomu cíli

### L2 Loops
- Ethernet narozdíl od IP (L3) nemá TTL
- Pokud vznikne fyzický okruh, frame v něm je navždy
#### Důsledky L2 loops
- Broadcast Storm - Broadcasty (např. ARP) se množí a síť se přehltí (vysokým přetížením CPU a paměti)
- Nestabilita MAC table - SW vidí stejnou MAC adresu přicházet z různých portů, jak rámec krouží dokola. Neustále přepisuje tabulku a neví, kam data poslat
- Duplicitní frame - Cílové PC dostanou stejný frame několikrát

# STA (Spanning Tree Algorithm)
- STP vytvoří logickou mapu sítě ve tvaru stromu, kde existuje jen jedna aktivní cesta:
1. Výběr Root Bridge - SW si zvolí bosse (s nejnižším Bridge ID)
2. Blokování portů - STP strategicky vybere porty, které způsobují smyčku a zablokuje je
3. Rekonfigurace - Pokud aktivní linka selže, STP porty odblokuje a začne posílat data záložní cestou
